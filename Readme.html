Documentación del Proyecto PHP

Descripción General

Este proyecto es un sistema de gestión de proyectos diseñado para registrar proyectos, gestionar usuarios y permitir la obtención de resultados. Está destinado a facilitar la administración y seguimiento de proyectos en una organización, proporcionando una plataforma centralizada para el manejo de diferentes roles de usuario y sus respectivas funciones.

Funcionalidades Principales
Registro de Proyectos: Permite a los usuarios registrar nuevos proyectos, incluyendo detalles importantes como el título, descripción, fechas y otros datos relevantes.

Gestión de Usuarios: Maneja diferentes roles de usuario (administrador, coordinador, evaluador, estudiante), cada uno con sus propias interfaces y permisos específicos para realizar acciones dentro del sistema.

Obtención de Resultados: Proporciona herramientas para evaluar y obtener resultados de los proyectos registrados, facilitando el seguimiento del progreso y la toma de decisiones basada en datos.

Roles de Usuario
Administrador: Tiene acceso completo al sistema, pudiendo gestionar usuarios, proyectos y realizar configuraciones generales.
Coordinador: Encargado de supervisar y coordinar los proyectos, además de tener acceso a funcionalidades específicas para su rol.
Evaluador: Responsable de evaluar los proyectos, proporcionando retroalimentación y resultados.
Estudiante: Puede registrar y gestionar sus propios proyectos, además de consultar los resultados de sus evaluaciones.
Componentes del Sistema
Interfaces de Usuario: Desarrolladas en JavaScript, con archivos específicos para cada rol (adminInterface.js, coordinador.js, evaluador.js, student.js).
Lógica del Servidor: Implementada en PHP, con archivos dedicados para manejar las acciones de cada rol y las interacciones con la base de datos (administrador.php, coordinador.php, evaluador.php, estudiante.php).
Páginas de Vistas: Archivos PHP que generan la interfaz visual del sistema, como la página de inicio de sesión (login.php).
Estilos y Recursos: Archivos CSS e imágenes utilizados para el diseño y apariencia del sistema.
Beneficios
Centralización: Todo el manejo de proyectos y usuarios se realiza en una plataforma única.
Eficiencia: Facilita la administración de proyectos y la evaluación de resultados de manera rápida y ordenada.
Accesibilidad: Diferentes roles de usuario pueden acceder y utilizar el sistema según sus permisos, mejorando la colaboración y coordinación.

Descripción de Archivos Principales

index.php

Archivo principal que se carga al acceder a la aplicación. Este archivo generalmente maneja las rutas y las vistas principales.

app.php

Archivo principal de la aplicación dentro de la carpeta includes. Este archivo contiene la lógica principal de la aplicación y maneja la inclusión de otros archivos necesarios.

<?php
try {
    if (!@include_once('includes/app.php')) throw new Exception('Opps, estamos presentando inconvenientes');
} catch (Exception $ex) {
    $errorPage = file_get_contents('includes/error.html');
    echo str_replace("{ERROR_MESSAGE}", $ex->getMessage(), $errorPage);
}
?>

Descripción del Código:

  • Bloque try: Intenta incluir el archivo app.php desde el directorio includes. Si falla, lanza una excepción con un mensaje de error.
  • Bloque catch: Captura la excepción lanzada y carga la página de error desde error.html, reemplazando el marcador {ERROR_MESSAGE} con el mensaje de la excepción.
